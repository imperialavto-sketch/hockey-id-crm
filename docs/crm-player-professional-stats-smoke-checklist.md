# CRM: ручная проверка карточки игрока — Hockey ID (professional stats)

Короткий сценарий для `/players/[id]` после входа под ролью с правом **players: view** (и **players: edit** для ручной записи).

## 1. Пустое состояние

1. Откройте карточку игрока без событий/наблюдений в Hockey ID.
2. Перейдите на вкладку **Hockey ID**.
3. **Ожидание:** краткий индикатор загрузки, затем заголовок вроде «Hockey ID — пока пусто» и карточка с подсказкой (для роли с правом редактирования — про блок «Ручная запись»; для только просмотра — без отсылки к форме выше).

## 2. Сохранение события (gameEvent)

1. На вкладке **Hockey ID** откройте **Ручная запись** → вкладка **Событие**.
2. Выберите тип (например «Гол»), при желании значение и заметку.
3. Нажмите **Сохранить** один раз.
4. **Ожидание:** кнопка в состоянии «Сохранение…», затем зелёный текст «Запись добавлена в сводку»; в сводке ниже появляется секция событий с учётом записи (без перезагрузки страницы).

## 3. Сохранение поведения (behavior)

1. **Ручная запись** → **Поведение**.
2. **Ожидание:** кнопка **Сохранить** неактивна, пока поле наблюдения пустое.
3. Заполните наблюдение (обязательно), при желании интенсивность.
4. Сохраните.
5. **Ожидание:** успех как в п.2; в блоке «Недавние наблюдения поведения» видна новая строка.

## 4. Обновление сводки без reload

1. После успешного сохранения события или поведения дождитесь строки «Обновляем данные…» (если показывается) и проверьте, что цифры/строки в сводке соответствуют только что внесённым данным.
2. **Ожидание:** полная перезагрузка страницы не требуется.

## 5. Ошибка API и повтор

1. Имитация: отключите сеть после загрузки вкладки или вызовите недоступный бэкенд; нажмите **Повторить** в карточке ошибки (или снова откройте вкладку после восстановления сети).
2. **Ожидание:** понятный текст ошибки; после успешного refetch ошибка снимается. Если данные уже были загружены, при сбое обновления остаётся последняя сводка и красная плашка с **Повторить**.

## 6. Переключение между игроками

1. На вкладке **Hockey ID** у игрока A с данными откройте сводку.
2. Перейдите к игроку B (тот же браузер, навигация CRM).
3. **Ожидание:** не показывается сводка игрока A; идёт загрузка, затем данные B (или пустое состояние для B). Быстрое переключение A ↔ B не оставляет «залипший» спиннер после ухода с вкладки.

## 7. Двойной клик по Сохранить

1. Быстро дважды нажмите **Сохранить** на событии.
2. **Ожидание:** одна запись (нет двойного создания из-за гонки в UI).
