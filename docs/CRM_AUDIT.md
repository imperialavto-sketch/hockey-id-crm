# Аудит CRM школы Hockey ID

**Дата:** март 2025  
**Цель:** Привести hockey-id-crm к чистой модульной структуре школы и отделить логику будущего приложения родителей.

---

## 1. Текущая структура

### 1.1 Страницы (app)

| Путь | Назначение | CRM / Parent |
|------|------------|--------------|
| `/dashboard` | Главная, KPI, ближайшие тренировки, активность | **CRM** |
| `/schools` | Школы (список, создание, карточка) | **CRM** |
| `/teams` | Команды | **CRM** |
| `/players` | Игроки (список, карточка, редактирование) | **CRM** |
| `/coaches` | Тренеры | **CRM** |
| `/schedule` | Расписание | **CRM** |
| `/trainings` | Тренировки (список, создание, детали) | **CRM** |
| `/finance` | Финансы (оплаты, задолженности) | **CRM** |
| `/analytics` | Аналитика | **CRM** |
| `/communications` | Коммуникации | **CRM** |
| `/settings` | Настройки, пользователи, роли, права | **CRM** |
| `/player/[id]` | Паспорт хоккеиста (графики, PDF, календарь травм) | **Shared** (CRM + Parent App) |
| `/parent` | Интерфейс родителя (дети, паспорт, оплаты, Stripe) | **Future Parent App** |
| `/payments` | Список оплат (legacy) | **Legacy** → редирект на `/finance` |
| `/coach` | Упрощённый портал тренера (тренировки, добавление) | **CRM** (COACH role) |

### 1.2 Навигация

- **CRM навигация** (для SCHOOL_ADMIN, MAIN_COACH, COACH, SCHOOL_MANAGER):  
  Главная, Школа, Команды, Игроки, Тренеры, Расписание, Финансы, Аналитика, Коммуникации, Настройки.
- **Parent навигация** (для PARENT):  
  Только «Мои дети» → `/parent`.
- `/parent` не включён в основную CRM-навигацию.
- `/payments` перенаправляется на `/finance`.

### 1.3 API

| Путь | Назначение | CRM / Parent |
|------|------------|--------------|
| `/api/schools/*` | Школы | CRM |
| `/api/teams/*` | Команды | CRM |
| `/api/coaches/*` | Тренеры | CRM |
| `/api/players/*` | Игроки | CRM |
| `/api/player/[id]/*` | Паспорт, навыки, достижения, видео, оплаты | Shared |
| `/api/trainings/*` | Тренировки | CRM |
| `/api/coach/trainings` | Тренировки тренера (для /coach) | CRM |
| `/api/payments/*` | Оплаты, Stripe checkout | CRM + Parent (Stripe redirect) |
| `/api/parent/players` | Игроки родителя | **Future Parent App** |
| `/api/dashboard/*` | Сводка, активности | CRM |
| `/api/analytics/*` | Аналитика | CRM |
| `/api/notifications/*` | Уведомления | Shared |

## 2. Prisma schema

### 2.1 Соответствие архитектуре

Основные сущности для CRM школы присутствуют:

- **User**, **UserRole** — SCHOOL_ADMIN, MAIN_COACH, COACH, SCHOOL_MANAGER, PARENT.
- **School**, **Team**, **Coach**, **Player**, **Parent** — корректно.
- **Role**, **Permission** — RBAC.
- **Training**, **Attendance** — расписание и посещаемость.
- **PlayerPayment**, **PlayerProfile**, **Passport**, **Skills**, **Medical**, **Achievement**, **Video**, **CoachRating**, **TrainingJournal** — поддерживают полный функционал.
- **Notification**, **Message** — коммуникации.
- **ActivityLog** — аудит.

### 2.2 Рекомендации

- Связь **User** → **Role** (многие ко многим) может потребоваться, если в будущем будут детальные роли через таблицу Role. Сейчас `User.role` — enum.
- **Coach** и **User** не связаны — тренер может быть как Coach, так и User. Для единого логина тренера рассмотреть связь Coach ↔ User.

## 3. RBAC (Role-Based Access Control)

### 3.1 Роли

| Роль | Описание |
|------|----------|
| SCHOOL_ADMIN | Полный доступ |
| MAIN_COACH | Команды, тренеры, расписание |
| COACH | Свои команды, тренировки, оценки |
| SCHOOL_MANAGER | Игроки, расписание, коммуникации, финансы |
| PARENT | Только просмотр своих детей → `/parent` |

### 3.2 Реализовано

- Фильтрация навигации по роли: PARENT видит только «Мои дети».
- Редирект PARENT с `/dashboard` на `/parent`.
- Таблица **Role** и **Permission** — готова для детальных прав (permissions matrix в Настройках).

### 3.3 Что доработать

- Middleware или проверки в API для фильтрации по роли и schoolId.
- Проверки доступа для COACH — только свои команды.
- Связь User.role с таблицей Role (если потребуется гибкая модель прав).
